One Time Pad Messenger

Information-theoretically secure message encryption using the One Time Pad cipher. When used correctly, OTP provides unconditional security — unbreakable regardless of adversary computing power.

🔒

Encrypt a Message

Enter your plaintext. A cryptographically random key is generated server-side using the OS CSPRNG and XOR'd with your message bytes. The key is displayed once and never stored.

Decrypt a Message

Provide both the Base64-encoded ciphertext and the corresponding OTP key you received from the sender. The key must match the ciphertext exactly — one byte per byte.

Security Notice
  • The OTP key is generated server-side via a FIPS-compliant CSPRNG (os.urandom).
  • Keys are never stored — copy them immediately after generation.
  • Each key must be used only once. Reuse completely compromises security.
  • Distribute key material through a separate, approved secure channel.
  • This tool is accessed exclusively over TLS 1.2/1.3 with NIST SP 800-52 cipher suites.
